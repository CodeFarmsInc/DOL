20. REVISION HISTORY

For various reasons, you may want to know the differences between individual versions of the software. Anybody who is running on a version older than 4.8 should request an update. Here are updates since Ver.4.9:

VERSION 5.0

Backward compatibility

No changes, fully compatible.

Corrections

No errors reported, only new features added.

Improvements

Function del() for the HASH organization has been improved so that it detects situations when an object seems to be in a hash table, but when you ask for it, it is not there.

The performance of the PAGER has been improved by keeping the track of dirty pages, and flushing only dirty pages to disk when close() is called. The !!TODO - Ask Jiri operation is also faster due to the use of memcpy() internally when copying large blocks of bytes.

New Features

  1. Function newFun() allows you to switch between different hashing functions while running your program.
  2. An ARRAY can be treated as a sorted collection, adding and removing items while maintaining the array ordered (sorted).

A new function etOrd(), performs binary search on the sorted array.

New functions available for PAGER: fill() returns the highest byte address currently used by the PAGER; flush() flushes all dirty pages to the disk.
